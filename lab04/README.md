Цей Python-проєкт реалізує **спрощену модель цифрового підпису**, що використовує персональні дані користувача для генерації ключів.
Проєкт розроблений у навчальних цілях та демонструє основні принципи роботи цифрових підписів:

* генерація ключів на основі персональної інформації;
* створення хешу документу;
* підпис документу;
* перевірка справжності підпису;
* демонстрація підробки.

Можливості системи

* Генерація пари **приватного та публічного ключів** на основі:

  * прізвища,
  * дати народження,
  * секретного слова.
* Використання SHA-256 для хешування даних.
* Підпис документу шляхом XOR-операції.
* Перевірка оригінальності підпису.
* Демонстрація підробки: зміна тексту → автоматично робить підпис недійсним.

## Технічні деталі

### **1. Генерація ключів**

Користувач вводить персональні дані, які об’єднуються в рядок.
Виконується хешування SHA-256 → результат перетворюється на приватний ключ.

Публічний ключ = `private_key * 7 mod MODULUS`.

### **2. Підпис документу**

1. Текст документа хешується через SHA-256.
2. Хеш переводиться у число.
3. Підпис = `hash XOR private_key`.

### **3. Перевірка підпису**

Перевіряється:

* чи співпадає хеш документу з оригінальним;
* чи правильно відновлюється хеш із підпису.

У разі зміни одного символу документу підпис стане недійсним.
